# Hank's Academic Writing Style

適用於中文學術論文（LaTeX），特別是 NCCU 碩士論文。

## 核心原則

**先說為什麼，再帶出術語。** 不要讓技術名詞突然出現，要先用白話說清楚「這個東西在做什麼」，再引入正式術語或數學式。讓數學式自己說話，不要在文字中重複解釋數學式已經表達的事。

## 句子結構

- 用連接詞引導邏輯流：「由於⋯因此」、「相較之下」、「相反地」、「更重要的是」
- 先說情境/問題，再說方法/理由，最後收攏到實用意義
- 技術術語出現時，附上括號英文或簡短說明，但不要另起一段大段解釋
- 避免「非缺失」、「可觀測」之類加了限定詞但語義明顯的冗詞，直接說「量測」就好

## 引入新概念的節奏

1. 先說這個概念「做什麼用」（功能導向）
2. 再說它的結構是什麼（形式導向）
3. 最後說選它的理由（比較導向）

**不好的範例：**
> Local Level Model 以隨機漫步（random walk）描述潛在品質水準的演化，$s_{t+1} = s_t + w_t$，觀測方程則將品質量測表示為潛在狀態與外生製程特徵的線性組合。

**好的範例：**
> 模型僅包含一條描述潛在品質水準如何從前一時點演化至下一時點的狀態方程，以及一條將品質量測連結至潛在狀態與外生製程特徵的觀測方程，不加入任何額外趨勢項或週期項。

## 選擇模型 / 方法的陳述方式

說明「為何選某個方法」時，核心理由要放在最前面，不要把功能特性當成選擇理由。

- 選擇 Local Level Model 的理由 → 「架構最貼近標準 Kalman filter」，而不是「能捕捉緩慢漂移」
- 理由說清楚後，再補充這個方法能做什麼

## 段落收尾

每段結尾通常收攏到一個明確的實用結論：「因此後續⋯採用⋯」、「此一設計使⋯不受⋯干擾」。避免開放式結尾。

## 數學符號出現後的文字說明

數學式之後的文字解釋，只說「非顯而易見」的部分：
- 說清楚下標的含義（$t \in k$ 代表什麼）
- 說清楚哪些項是隨機的、哪些是確定的
- 不要重複數學式已經寫明的恆等關係
